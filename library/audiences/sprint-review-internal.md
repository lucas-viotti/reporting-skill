# Audience: Sprint Review Internal

For biweekly sprint updates delivered to program management, delivery managers,
and internal stakeholders following sprint execution reviews.

---

## Style

`concise-professional`

## Template

`milestone-summary`

---

## Include

- Active milestones with updates this sprint
- Challenges and risks with path-to-green details
- Sprint-level status signal if any milestone is at risk

## Exclude

- Technical implementation detail (APIs, services, configurations)
- Geography references outside the US (unless config explicitly enables multi-geo)
- Milestones with no activity this sprint
- Team-internal operational context

## Constraints

- Maximum 1-2 sentences per milestone
- Use business outcomes language throughout
- "Challenges & Path to Green" section only when active risks exist

---

## Notes

This audience profile is designed for the Project Troy / Destination Architecture biweekly
execution review. It assumes US Credit Card delivery context.

To adapt for other projects: update `include`/`exclude` constraints in a project-specific
audience file, or override via `report-config.md`.
