# Template: Executive Bullets

Use for C-level / VP audiences. 3-5 bullets maximum. Risk-first framing.
Pairs with the `executive` tone.

---

## Output Structure

```
### [DATE RANGE] Update

[Overall status — 1 sentence. Lead with risk if present.]

- **[Milestone]**: [Status + risk/gate if applicable]
- **[Milestone]**: [Status + risk/gate if applicable]
- **[Milestone]**: [Status + risk/gate if applicable]

[Escalation or ask — only if action needed from audience]
```

---

## Rules

- Maximum **5 bullets** — pick the most important milestones.
- **Lead with risk** — put at-risk items first.
- **One sentence per bullet** — no sub-bullets.
- Omit the escalation line if no decision or action is needed.
- Use executive tone — no team names, service names, or engineering terms.

---

## Real Example

```
### Feb 23 - Mar 6, 2026

Beta timeline at risk — Revolving and Lateness delayed to May; no other milestones affected.

- **Revolving**: At risk — May target (was April). Dedicated team in place; design finalized.
- **Lateness**: At risk — implementation starting next sprint.
- **Billing (Close complete bill)**: On track; staging validation underway.
- **Reversals**: Authorization reversal development complete; validations in progress.

No executive action needed this sprint. Updated roadmap publishing Feb 27.
```
